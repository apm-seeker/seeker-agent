# seeker-agent

## 커밋 메시지 규칙

```
[커밋 종류]: 커밋 메시지
```

| 종류 | 설명 |
|---|---|
| `feat` | 새로운 기능 추가 |
| `refact` | 코드 리팩토링 |
| `enhancement` | 기존 기능 개선 |
| `fix` | 버그 수정 |
| `docs` | 문서 작업 |

- 커밋 메시지는 한글 사용 가능
- 예시: `feat: 트레이스 생성 기능 추가`

## 브랜치 규칙

```
[브랜치 종류]/#issueNumber/작업_내용
```

| 종류 | 설명 |
|---|---|
| `feat` | 새로운 기능 추가 |
| `refact` | 코드 리팩토링 |
| `enhancement` | 기존 기능 개선 |
| `fix` | 버그 수정 |
| `docs` | 문서 작업 |

- 작업 내용은 영어, snake_case로 작성
- 예시: `feat/#1/create_trace`
